Post dedicado à implementação e uso do pipenv.

Bom, primeiro é interessando que explique o que se trata a ferramenta chamada pipenv, que é, em essência, utilizada para para criar um ambiente virtual e um utilitário para instalação de pacotes, gerenciamento de ambientes virtuais (pyenv) e possui todos os comandos associados às bibliotecas utilizadas.

Basicamente, o pipenv oferece ferramentas similiares ao pip e a virtualenv. E, como benefício explícito em relação às duas ferramentas, verifica-se que o pipenv não sofre com problemas com eventuais atualizações da lingugem python, como a descrita pelo Arth Tyagi no seu post

O próprio fato de você não neceissatar do uso do pip e da virtualenv separadamente, é um grande trunfo do Pipenv, além disso, a ferramenta fornece uma árvore de dependências (dependency graph ), para melhor controle de dependências do projeto.

O Uso do Pipenv

Primeiro e antes de tudo, vamos começar lendo a documentação

.

A partir disso selecioneu alguns comandos básicos para se trabalhar com o pipenv

Instalar o Pipenv


pip3(ou pip) install pipenv

Para ativar o enviroment:


pipenv shell

Exemplo como instalar o pacote no ambiente

pipenv install **package_name** pipenv install django

Através do pipenv é possível executar os comandos pip, como por exemplo:


pipenv run pip freeze
pipenv run pip freeze > requirements.txt

Do mesmo modo, é possível instalar os requeriments


pipenv install -r requirements.txt
